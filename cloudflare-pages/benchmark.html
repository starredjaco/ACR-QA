Why This Benchmark Is Different

Pre-registered, signed, continuously re-run

1 Pre-registered methodology — the scoring formula and inclusion criteria are committed to git before any tool is run. No cherry-picking.
2 Third-party ground truth — CWE labels come from SecurityEval dataset authors, not ACR-QA. We have no control over what counts as a vulnerability.
3 Same corpus, same conditions — Bandit and Semgrep run on the same files, same commit, same machine as ACR-QA. No tilted playing field.
4 CI-signed output — weekly GitHub Actions job re-runs the benchmark, signs the result JSON with ACR-QA's ECDSA key, and publishes to Rekor. Anyone can verify.
5 Reproducible in 2 commands — clone the dataset, run the script. Results are deterministic at the same tool versions.
Reproduce It Yourself

Two commands, fully open

git clone https://github.com/s2e-lab/SecurityEval TESTS/evaluation/securityeval
python3 scripts/run_benchmark_p2.py --dataset-dir TESTS/evaluation/securityeval
